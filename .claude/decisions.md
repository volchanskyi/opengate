## ADR-001: Wire Protocol — MessagePack + Internally Tagged Enums

**Status**: Accepted | **Phase**: 1

**Decision**: MessagePack encoding with `#[serde(tag = "type")]` internally tagged enums in Rust. Go uses `vmihailenco/msgpack` with matching struct tags.

**Frame format**: `[1-byte type][4-byte BE length][payload]`; Ping/Pong are single-byte (no payload). Handshake uses raw binary encoding (not msgpack), type bytes `0x10–0x15`.

**Why**: JSON is too verbose; protobuf requires schema compilation; custom binary is maintenance-heavy. MessagePack is compact, schema-free, and has good Rust+Go library support.

**Consequences**: Cross-language compat verified in CI via `make golden` (golden files in `/testdata/golden/`, generated by Rust, verified by Go).

## ADR-002: Golden File Tests for Cross-Language Compatibility

**Status**: Accepted | **Phase**: 1

**Decision**: Rust generates canonical binary fixtures ("golden files") in `/testdata/golden/`. Go integration tests load and deserialize them to verify byte-for-byte compatibility.

**Why**: Two independent implementations of the same protocol will silently diverge without a shared contract test.

**Consequences**: Run `make golden` after any protocol change. Go→Rust direction not yet covered (see `techdebt.md`). Golden files are committed.

## ADR-003: SQLite + WAL Mode via Pure Go Driver

**Status**: Accepted | **Phase**: 2

**Decision**: `modernc.org/sqlite` (pure Go, no CGo), WAL mode, `foreign_keys` pragma, `MaxOpenConns(1)`.

**Why**: No external DB dependency; pure Go avoids CGo issues for aarch64-musl cross-compilation (OCI ARM64 deployment).

**Consequences**: Single writer at a time (see `techdebt.md`). Phase 13 will replace with PostgreSQL.

## ADR-004: ECDSA P-256 mTLS with Self-Signed CA + CSR Enrollment

**Status**: Accepted | **Phase**: 2

**Decision**: Server is its own CA (10yr P-256 cert). Agents enroll via PKCS#10 CSR posted to `/api/v1/enroll/{token}`; server signs and returns `cert_pem + ca_pem`. TLS 1.3 minimum. `devices.group_id` is NULLABLE with ON DELETE SET NULL — agents register without a group.

**Why**: Mutual authentication without a public PKI dependency. Agents run on untrusted networks.

**Consequences**: CA key loss = full re-enrollment. One-time tokens required for enrollment (issued by admin).

## ADR-005: QUIC mTLS via quic-go (Server Opens Control Stream — Workaround)

**Status**: Accepted (workaround in place) | **Phase**: 4

**Decision**: `quic-go v0.48` for agent transport. Due to a quic-go bug where `AcceptStream` doesn't work with mTLS client certs, the **server opens the control stream** instead of the agent.

**Why**: QUIC gives multiplexed, low-latency transport with no head-of-line blocking.

**Consequences**: Workaround holds up to ~20k agents. Must revert when quic-go fixes mTLS `AcceptStream`. See `techdebt.md` and [plans/quic-stream-ownership-fix.md](plans/quic-stream-ownership-fix.md).

## ADR-006: Platform Traits with Null Implementations for Headless Environments

**Status**: Accepted | **Phase**: 5

**Decision**: Agent capabilities are Rust traits (`ScreenCapture` with `async_trait`, `InputInjector` and `ServiceLifecycle` as sync/object-safe). Null implementations used in CI and headless containers.

**Why**: Agent must not panic on missing display/input hardware.

**Consequences**: `async_trait` adds a small per-call allocation for screen capture (acceptable at ~30fps). Windows platform is currently all stubs (see `techdebt.md`).

## ADR-007: VAPID Web Push with Keys Persisted in dataDir

**Status**: Accepted | **Phase**: 10

**Decision**: Server generates VAPID keypair on first run, persists to `{dataDir}/vapid.json`. `Notifier` interface decouples push logic from HTTP handlers.

**Why**: Standard Web Push without a third-party push service dependency.

**Consequences**: Changing VAPID keys invalidates all existing browser subscriptions. `dataDir` must be a Docker volume mount.

## ADR-008: aarch64-unknown-linux-musl Cross-Compilation via `cross`

**Status**: Accepted | **Phase**: CD-C

**Decision**: `cross` tool (Docker-based) for Rust → `aarch64-unknown-linux-musl`. Go uses `GOARCH=arm64` natively. Produces fully static binaries.

**Why**: Production target is Oracle Cloud ARM64 Always Free. CI runners are x86_64.

**Consequences**: Docker daemon required in CI. SELinux systems need `restorecon` after OTA binary replacement (fixed 2026-03-26 in `update.rs`).
