# SonarCloud Quality Gate — Implementation Plan

## Context

The CI has 12 jobs in `ci.yml` with auto-merge gating. Coverage exists only for Go (70% threshold via `coverage.out`). No SonarCloud/SonarQube today — CodeQL handles SAST. Goal: add SonarCloud with aggressive quality gates using official SonarSource GitHub Actions. This is free for public repos, provides PR decoration, and tracks technical debt over time.

---

## Step 1: Prerequisites (manual, outside code)

1. Sign up at sonarcloud.io → import `volchanskyi` GitHub org → create `opengate` project
2. Organization key: `volchanskyi`, Project key: `volchanskyi`
3. Add `SONAR_TOKEN` secret to repo Settings → Secrets → Actions

## Step 2: Create `sonar-project.properties`

**New file:** `sonar-project.properties` (repo root)

```properties
sonar.organization=volchanskyi
sonar.projectKey=volchanskyi
sonar.projectName=OpenGate

# Sources
sonar.sources=server/internal,agent/crates,web/src
sonar.tests=server/internal,server/tests,agent/crates,web/src
sonar.test.inclusions=**/*_test.go,**/tests/**,**/*.test.ts,**/*.test.tsx,**/*.spec.ts

# Go coverage (generated by go-unit job)
sonar.go.coverage.reportPaths=server/coverage.out

# TypeScript/JS coverage (generated by web job)
sonar.javascript.lcov.reportPaths=web/coverage/lcov.info

# Rust — generic coverage import (llvm-cov lcov output)
sonar.coverageReportPaths=agent/lcov.info

# Exclusions
sonar.exclusions=**/testutil/**,**/testdata/**,**/node_modules/**,**/target/**,**/*.test.ts,**/*.test.tsx,**/*_test.go
sonar.coverage.exclusions=**/testutil/**,**/testdata/**,**/*_test.go,**/*.test.ts,**/*.test.tsx
```

## Step 3: Add coverage generation to existing test jobs

### 3a. Go coverage — already exists
- **File:** `.github/workflows/ci.yml`, job `go-unit`
- `coverage.out` is already generated. Add artifact upload step after the existing coverage check:
  ```yaml
  - name: Upload Go coverage
    uses: actions/upload-artifact@v4
    with:
      name: go-coverage
      path: server/coverage.out
  ```

### 3b. Web coverage — add to `web` job
- **File:** `.github/workflows/ci.yml`, job `web`
- Install coverage provider and modify vitest command:
  ```yaml
  - name: Install coverage provider
    run: npm install -D @vitest/coverage-v8
  - name: Test
    run: npx vitest run --coverage --reporter=verbose ...
  ```
- **File:** `web/vitest.config.ts` — add coverage config:
  ```ts
  coverage: {
    provider: 'v8',
    reporter: ['lcov', 'text'],
    reportsDirectory: './coverage',
  }
  ```
- Upload artifact:
  ```yaml
  - name: Upload Web coverage
    uses: actions/upload-artifact@v4
    with:
      name: web-coverage
      path: web/coverage/lcov.info
  ```

### 3c. Rust coverage — add to `rust-test` job
- **File:** `.github/workflows/ci.yml`, job `rust-test`
- Add `cargo-llvm-cov` step after existing tests:
  ```yaml
  - name: Install cargo-llvm-cov
    uses: taiki-e/install-action@cargo-llvm-cov
  - name: Generate Rust coverage
    run: cargo llvm-cov --workspace --lcov --output-path lcov.info
  - name: Upload Rust coverage
    uses: actions/upload-artifact@v4
    with:
      name: rust-coverage
      path: agent/lcov.info
  ```

## Step 4: Add `sonarcloud` job to `ci.yml`

**File:** `.github/workflows/ci.yml`

New job, placed after test jobs:

```yaml
sonarcloud:
  name: SonarCloud Analysis
  runs-on: ubuntu-latest
  needs: [go-unit, rust-test, web]
  if: github.event_name != 'schedule'
  steps:
    - uses: actions/checkout@v4
      with:
        fetch-depth: 0  # Full history for blame/new code detection

    - name: Download Go coverage
      uses: actions/download-artifact@v4
      with:
        name: go-coverage
        path: server/

    - name: Download Web coverage
      uses: actions/download-artifact@v4
      with:
        name: web-coverage
        path: web/coverage/

    - name: Download Rust coverage
      uses: actions/download-artifact@v4
      with:
        name: rust-coverage
        path: agent/

    - name: SonarCloud Scan
      uses: SonarSource/sonarqube-scan-action@v5
      env:
        SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
```

## Step 6: Configure quality gate in SonarCloud UI

Set these thresholds on the project's quality gate (SonarCloud web UI):

| Metric | Condition | Value |
|--------|-----------|-------|
| New code coverage | ≥ | 80% |
| Overall coverage | ≥ | 70% |
| New bugs | = | 0 |
| New vulnerabilities | = | 0 |
| New code smells | = | 0 |
| Reliability rating on new code | = | A (no new bugs) |
| Security rating on new code | = | A |
| Maintainability rating on new code | = | A |
| New security hotspots reviewed | = | 100% |
| Duplicated lines on new code | < | 3% |
