# Micro-Plan: Bump web TypeScript to 6.x (when upstream allows)

**Register entry:** [techdebt.md](../techdebt.md) — "`web/package.json` TypeScript
pinned to ^5.9.3 — `openapi-typescript` peer conflict." **Master:**
`techdebt-paydown-master.md`. **Branch:** `dev`. **Owner:** web. **Status:** **blocked
upstream** — this is a watch + bump procedure, not ready-to-implement.

## 1. Problem

TypeScript is pinned `^5.9.3` in [`web/package.json`](../../web/package.json) because
`openapi-typescript@7.13.0` (used by `npm run generate:api`) declares
`peerDependencies: { typescript: "^5.x" }`. A clean `npm ci` (the `node:24-alpine`
Docker build) fails hard with `ERESOLVE` on TS 6 — reproduced. All other deps from
that Dependabot group landed; only the TS major stayed back.

## 2. Trigger (the precondition)

Proceed **only** once `openapi-typescript` ships a release whose `typescript`
peerDependencies range admits 6.x. Check:

```
npm view openapi-typescript version
npm view openapi-typescript peerDependencies
```

If the peer range includes `^6` (or `>=5`), the block is cleared and this plan is
actionable. Until then, leave the pin and this plan parked.

## 3. Scope

**In:** bump `typescript` and `openapi-typescript` together; regenerate the OpenAPI TS
client; fix TS6 breakages; prove `npm ci` is clean.
**Out:** unrelated dep bumps; runtime behavior changes.

## 4. File inventory

| File | Change |
|---|---|
| `web/package.json` | `typescript` → `^6.x`; `openapi-typescript` → the TS6-compatible release. |
| `web/package-lock.json` | Regenerated by `npm install`. |
| `web/src/types/api.d.ts` (or generated path) | Re-run `npm run generate:api`. |
| `web/**` | Any TS 6 compiler-break fixes (likely lib/strictness changes). |

## 5. Approach

1. Confirm the trigger (§2).
2. Bump both deps; `npm install`; **then `npm ci`** to prove the `ERESOLVE` case is
   gone (mirror the Docker build: `node:24-alpine`).
3. `npm run generate:api` — confirm codegen still works against the OpenAPI spec.
4. `tsc --noEmit` / `npm run build`; fix breakages.
5. Align `typescript-eslint` to a TS6-supporting version if needed; `npm run lint`.
6. `/precommit` → commit → `/refactor` → `/precommit` → commit → push.

## 6. Quality metrics / acceptance

- [ ] `npm ci` succeeds with TS 6 (the exact failure mode is gone).
- [ ] `npm run generate:api` produces unchanged-or-correct types.
- [ ] `tsc`/build/lint green; the `build-image` Docker build passes.
- [ ] `/precommit` green.

## 7. NFRs

- **Maintainability:** keeps the toolchain current; unblocks future TS6-only deps.
- **Security:** newer TS/openapi-typescript pick up upstream fixes; no new attack surface.

## 8. Reviewer checklist

- [ ] `npm ci` (not just `npm install`) verified clean.
- [ ] Generated API types reviewed for unexpected diffs.
- [ ] No `any` introduced to paper over TS6 strictness (strict-mode rule).

## 9. Risks

- TS 6 may introduce strictness breaks across the app; budget fix time.
- Until the trigger clears, do not force-resolve the peer (no lenient overrides in CI).
